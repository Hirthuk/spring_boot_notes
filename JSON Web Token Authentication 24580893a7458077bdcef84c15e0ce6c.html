JSON Web Token Authentication

✅ JWT & JJWT Concepts Used in the Code

🔐 What is JWT?

JWT (JSON Web Token) is a compact, secure way to transmit information between parties as a JSON object.

It is commonly used for authentication and authorization.

A JWT has 3 parts:

  1. Header → algorithm & type
  1. Payload (Claims) → data like username, role, expiry
  1. Signature → to verify token integrity

📦 Library: jjwt (io.jsonwebtoken)

  • A Java library for creating, signing, parsing, and verifying JWTs.
  • Supports various signing algorithms like HS256, RS256, etc.
  • The latest versions (v0.12+) offer a new, fluent and type-safe API.

🔑 Concepts Used in Code

🔐 SecretKey

  • Used to sign and verify the JWT.
  • HS256 requires a symmetric secret key (same key used for both signing & verification).
SecretKey key = Jwts.SIG.HS256.key().build();

🛠 Jwts.builder()

Used to create the token.

Common builder methods:

  • .subject(String) – sets the username or identifier
  • .issuedAt(Date) – time token was created
  • .expiration(Date) – sets token expiry
  • .claims(Map<String, Object>) – adds custom key–value pairs (e.g., role)
  • .signWith(Key, SignatureAlgorithm) – signs the token
  • .compact() – returns the final JWT as a String

📤 Claims

  • Claims are pieces of information in the token payload.
  • Two types:
    • Standard Claims → e.g., sub, iat, exp
    • Custom Claims → user-defined, e.g., "role": "admin"

In your code:

claims.put("role", role); // custom claim

🔎 Parsing Tokens

Jwts.parser()
    .verifyWith(SECRET_KEY)
    .build()
    .parseSignedClaims(token)
    .getPayload();
  • parser() → builds a parser
  • verifyWith(key) → checks if the signature is valid
  • parseSignedClaims(token) → parses the JWT and returns signed content
  • getPayload() → extracts claims from the JWT

🔁 Token Validation

Used to check:

  • User info: getSubject() → returns sub (username)
  • Custom info: get("role", String.class)
  • Expiration check:
    claims.getExpiration().before(new Date())
