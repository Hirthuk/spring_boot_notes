Swagger UI Implementation — InsiderHub Backend (Code-Level Deep Dive)

1. Purpose

Swagger UI lets you:

  • Auto-generate API documentation from your Spring Boot controllers.
  • Interactively test endpoints without Postman.
  • Integrate JWT authentication so you can call secured APIs from the browser.

We implemented it using Springdoc OpenAPI Starter for WebMVC (OpenAPI 3.x).

2. Step-by-Step Implementation

Step 1 — Maven Dependency

In pom.xml:

<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.5.0</version>
</dependency>

What happens under the hood:

  • Spring Boot’s spring.factories auto-config mechanism detects this starter.
  • It auto-registers:
    • /v3/api-docs → REST endpoint that returns OpenAPI spec JSON.
    • /swagger-ui/** → Static resource handler serving Swagger HTML, JS, CSS from the jar’s /META-INF/resources.
  • It scans all @RestController beans and Spring MVC mappings to build the spec dynamically at runtime.

Step 2 — Create OpenAPIConfig

File: com.insider.backend.configuration.OpenAPIConfig

@Configuration // Marks this class as a Spring config bean provider
public class OpenAPIConfig {

  // 1. Define JWT security scheme for Swagger UI
  private SecurityScheme bearerScheme() {
    return new SecurityScheme()
        .name("BearerAuth")             // Security name referenced in API spec
        .type(SecurityScheme.Type.HTTP) // Tells Swagger it's HTTP authentication
        .scheme("bearer")               // Token type is Bearer
        .bearerFormat("JWT");           // Just a label for display, not functional
  }

  // 2. Register the OpenAPI bean
  @Bean
  public OpenAPI openAPI() {
    return new OpenAPI()
        // a) Add our BearerAuth scheme to OpenAPI components
        .components(
            new Components().addSecuritySchemes("BearerAuth", bearerScheme())
        )
        // b) Apply security requirement globally (all endpoints need BearerAuth)
        .addSecurityItem(
            new SecurityRequirement().addList("BearerAuth")
        )
        // c) Set API metadata
        .info(new Info()
            .title("InsiderHub API")
            .version("1.0")
            .description("API documentation for InsiderHub backend")
            .contact(new Contact()
                .name("InsiderHub Team")
                .email("support@insiderhub.local")
            )
        );
  }
}

Step 3 — How It Works Internally

3.1 Bean Registration

  • @Configuration + @Bean → Spring registers an OpenAPI bean in the application context.
  • Springdoc detects this bean and merges your config with its auto-generated spec.

3.2 Security Scheme

  • bearerScheme() creates a Swagger SecurityScheme object.
  • "BearerAuth" is a key that appears in:
    • components.securitySchemes.BearerAuth
    • security: [{ BearerAuth: [] }]
  • Swagger UI sees this and adds an Authorize button in the top-right.

3.3 Global Security

.addSecurityItem(new SecurityRequirement().addList("BearerAuth"))
  • Tells OpenAPI: all endpoints in the doc require "BearerAuth" unless overridden.
  • Swagger UI automatically sends Authorization: Bearer <token> for secured requests when authorized.

4. Runtime Flow

  1. App Startup
    • Springdoc scans your controllers & request mappings (HandlerMethod list from Spring MVC).
    • Builds an OpenAPI object with:
      • Paths (URLs + HTTP methods)
      • Parameters
      • Request bodies
      • Responses
    • Merges your custom OpenAPIConfig settings.
  1. Access Swagger UI
    • Visit: http://localhost:8080/swagger-ui/index.html
    • Swagger UI frontend calls /v3/api-docs in the background to fetch the JSON spec.
    • Renders endpoints grouped by controller.
  1. Authorize for JWT
    • Click “Authorize” → Enter token without "Bearer " prefix.
    • Swagger JS stores it in memory and attaches it as Authorization: Bearer <token> on every API call.
  1. Make API Calls
    • Swagger sends AJAX calls to your backend endpoints.
    • Spring Security + JWT filter validate the request.

5. Security Integration in Your App

In SecurityFilterChain:

.requestMatchers(
    "/v3/api-docs/**",
    "/swagger-ui/**",
    "/swagger-ui.html"
).hasRole("ADMIN") // or .permitAll() for dev
  • If .hasRole("ADMIN"), only authenticated admin users can open Swagger UI.
  • If .permitAll(), it’s public (dev mode only).

6. Login and Testing in Swagger

  • Login API (/api/auth/login) is marked permitAll() in security.
  • Call it from Swagger → Receive JWT token.
  • Use Authorize → Enter token.
  • Now you can call all secured APIs directly from Swagger UI without manually adding headers.

7. URL Cheat Sheet

  • Swagger UI:
    http://localhost:8080/swagger-ui/index.html
  • OpenAPI spec JSON:
    http://localhost:8080/v3/api-docs
