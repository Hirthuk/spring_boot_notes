Entity classes

üîπ Entity Classes in Spring Boot (JPA)

  • Represent database tables in Java.
  • Annotated with @Entity and @Table.
  • Each field maps to a column in the table.
  • Used by JPA Repository to perform CRUD operations automatically.
  • No SQL required ‚Äî Spring handles DB interactions based on these classes.
  • We can add Validation, Index, Not Null, Unique kind of constraints with the annotation

JPA Repository

üîπ JPA Repository in Spring Boot

  • Interface used to perform CRUD operations on Entity classes.
  • Extends JpaRepository<EntityName, IdType>.
  • No need to write SQL or implementation logic.
  • Spring auto-generates methods like save(), findById(), delete(), findAll(), etc.
  • Can define custom methods using method names (e.g., findByEmail()).
JPA Methods
  • findAll() Returns all the table data in the List of Entity data type ‚Üí List<UserEntity>
  • save() Returns the UserEntity object as return Data type
  • delete() return void(nothing)
  • get() - It retrieves the entire row (record) from the database as a Java object (UserEntity), mapped from the table that your UserEntity class represents. It is used with the object of the UserEntity ‚Üí UserEntity user ‚Üí user.get() - Return type is Optional
