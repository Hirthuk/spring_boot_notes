Email Configuration

📓 Notes: Thymeleaf Email Template Setup in Spring Boot

1. Dependencies in pom.xml

We included:

  1. Thymeleaf for Template Rendering
    <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-thymeleaf</artifactId>
</dependency>
    • This lets us use .html templates with placeholders (${variable}) to dynamically insert data.
  1. JavaMail for Sending Emails
    <dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-mail</artifactId>
</dependency>
    • Enables JavaMailSender to send emails.
    • Works with SMTP settings (configured in application.properties).

2. TemplateEngine in Spring Boot

  • The TemplateEngine is the core Thymeleaf processor.
  • Spring Boot auto-configures a SpringTemplateEngine bean when spring-boot-starter-thymeleaf is in the classpath.
  • This engine:
    • Loads the .html file from src/main/resources/templates/
    • Replaces placeholders with real values using the Context.

Example:

String html = templateEngine.process("email-template", context);

Here:

  • "email-template" → refers to email-template.html
  • context → contains the variables for substitution.

3. Context in Thymeleaf

  • Context is like a map to store variables for the template.
  • Each variable in the context can be accessed in HTML with ${variableName}.

Example:

Context context = new Context();
context.setVariable("name", "Sharan");
context.setVariable("link", "https://example.com");

In the template:

<p>Hello, <b th:text="${name}"></b></p>
<a th:href="${link}">Click here</a>

This gets replaced by:

<p>Hello, <b>Sharan</b></p>
<a href="https://example.com">Click here</a>

4. Email Sending Flow

  1. Load Template
    • templateEngine.process("email-template", context) → generates HTML string.
  1. Create Email Message
    • MimeMessage → represents the email.
    • MimeMessageHelper → helps set subject, body, recipients.
  1. Send Email
    • javaMailSender.send(mimeMessage) → actually sends the email.

Example:

MimeMessage message = javaMailSender.createMimeMessage();
MimeMessageHelper helper = new MimeMessageHelper(message, true);

helper.setTo("admin@example.com");
helper.setSubject("Welcome Email");
helper.setText(htmlContent, true); // true → HTML content

javaMailSender.send(message);

5. How String[] toAddresses Fits

String[] toAddresses = adminEmails.stream()
    .map(UsersEntity::getEmail)
    .toArray(String[]::new);
  • We take a list of UsersEntity objects → extract email → put into a String[].
  • Used for sending to multiple recipients:
helper.setTo(toAddresses);

6. Template File Location

  • Path: src/main/resources/templates/email-template.html
  • Example:
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<body>
    <p>Hello <span th:text="${name}"></span>,</p>
    <p>Please click <a th:href="${link}">here</a> to verify.</p>
</body>
</html>

7. How It Works Together

  1. Context → store variables (name, link, etc.)
  1. TemplateEngine → processes HTML template and injects variables.
  1. JavaMailSender → sends processed HTML as an email.

✅ Summary Flow

Data → Context → Thymeleaf TemplateEngine → HTML String → MimeMessageHelper → JavaMailSender → Email Sent
